# ShellX Changelog

All notable changes to ShellX will be documented here.

---

## [4.0.0] – Planned
### Added
- Full modularization of ShellX into standalone engine
- Plugin system for workers and pipelines
- Cloud-based LinkX timeline
- Stone diff viewer UI
- VS Code extension with interactive panels
- ShellX Pro (commercial) support

### Changed
- Complete separation from KickerOS core
- New command parser with strict mode
- Unified worker interface

---

## [3.0.0] – 2026-04-01
### Added
- Cognitive Reporting Engine (V3)
- `shellx --xx` command for intent-aware reporting
- Collector → Merger → Cognitive Compressor pipeline
- Full report generator with `.reports/` output
- LinkX integration for summary + diff tracking
- Stone creation for recovery snapshots
- Safe-mode autofix (no destructive writes)
- New project structure for ShellX modules
- Initial VS Code extension manifest
- Documentation folder and architecture diagram

### Changed
- Updated command handler to support V3 cognitive mode
- Improved worker isolation for deep audit, debug, architect, memory

### Fixed
- Report overwriting issues
- Missing sections in multi-agent pipelines
- Inconsistent summary output

---

## [2.0.0] – Historical
- Multi-agent system introduced
- Superman mode added
- LinkX + Stones added

---

## [1.0.0] – Historical
- Original ShellX report generator
- Basic command system
