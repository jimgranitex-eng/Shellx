# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 3.x     | ✅ Active support |
| 2.x     | ⚠️ Security fixes only |
| 1.x     | ❌ No longer supported |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub Issues.**

If you discover a security vulnerability in ShellX, please report it responsibly:

1. **Open a private security advisory** via [GitHub Security Advisories](https://github.com/jimgranitex-eng/Shellx/security/advisories/new).
2. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

---

## Response Timeline

| Action | Timeframe |
|--------|-----------|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 7 days |
| Fix or mitigation | Within 30 days (depending on severity) |
| Public disclosure | After fix is released |

---

## Scope

This security policy covers:

- ShellX core engine
- ShellX workers
- ShellX CLI commands
- Activation and licensing system (ShellX Pro)

---

## Out of Scope

- Third-party dependencies (report to the respective project)
- Issues in forks or unofficial distributions

---

## Security Best Practices

When using ShellX:

- Keep your installation up to date.
- Do not expose license keys or credentials in logs or public repositories.
- Review the code of any custom workers before running them.

---

Thank you for helping keep ShellX secure. 🔐
