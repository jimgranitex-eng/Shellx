Quickstart Guide

Get up and running with ShellX in under 5 minutes.

Prerequisites

  • Node.js 18+
  • Git
  • A terminal

Installation

Option 1 — Clone from GitHub

git clone https://github.com/jimgranitex-eng/Shellx.git
cd Shellx
npm install

Option 2 — Global install (coming soon)

npm install -g shellx

V1 — Simple Report

The original ShellX interface. Pass a message and get a structured report.

$ shellx "Fix the shader logic and update the color pipeline."

What it does:

  • Reads your message
  • Executes the task
  • Generates a timestamped report
  • Saves it to your project

V2 — Superman Mode

All-systems diagnostic engine. Runs the complete 9-worker pipeline.

$ shellx-superman "audit everything and stabilize the engine"

What it does:

  1. Deep audit
  2. Architecture mapping
  3. Memory analysis
  4. Safe-mode autofix
  5. Verification pipeline
  6. Stone creation
  7. LinkX update

V3 — Cognitive Mode (Recommended)

The flagship natural language interface. Describe what you want, get results.

$ shellx --xx "Fix the rendering pipeline and update the shader logic."
🔍 Analyzing intent...
🧠 Running cognitive pipeline...
✅ Complete
1. What you wanted: Fix rendering pipeline & update shaders
2. What ShellX did: Deep audit → Debug → Autofix → Verify
3. What changed: 8 files modified, 1 stone created

The output is always a clean 1-2-3 summary. Human-readable. No noise.

Output Structure

Every run produces:

Output Location
Report reports/YYYY-MM-DD-HH-MM-SS.md
Stone snapshot stones/stone-YYYY-MM-DD.json
LinkX update linkx/timeline.json
