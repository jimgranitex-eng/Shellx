ShellX — Quickstart Guide

Get up and running with ShellX in under 5 minutes.

Prerequisites

  • Node.js 18+ (or your preferred runtime)
  • Git
  • A terminal

Installation

Option 1 — Global install (recommended)

$ npm install -g shellx-dev-cli
$ shellx linkx init
$ shellx report

Option 2 — Clone from GitHub

$ git clone https://github.com/jimgranitex-eng/Shellx.git
$ cd Shellx && npm install

Basic Usage

V1 — Simple Report

$ shellx "Fix the shader logic and update the color pipeline."

What it does: reads your message, executes the task, generates a timestamped report, saves it to your project.

V2 — Multi-Agent Diagnostic (Superman)

$ shellx-superman "audit everything and stabilize the engine"

Runs: deep audit, architecture mapping, memory analysis, safe-mode autofix, verification pipeline, stone creation, LinkX update.

V3 — Cognitive Mode (Recommended)

$ shellx --xx "Fix the rendering pipeline and update the shader logic."

1. What you wanted
2. What ShellX did
3. What changed

With checkmarks. Clean. Human-readable.

Output Structure

Every run produces:

Output Location
Reportreports/YYYY-MM-DD-HH-MM-SS.md
Stone snapshotstones/stone-YYYY-MM-DD.json
LinkX updatelinkx/timeline.json
